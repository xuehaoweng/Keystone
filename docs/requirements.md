# LLM Gateway 需求清单

## 目标

构建一个企业级大模型统一入口，解决多模型接入混乱、成本不可见、模型故障不可控、权限边界不清和资源错配问题。

## 已实现

| 模块 | 能力 | 状态 |
|------|------|------|
| 统一入口 | 第三方业务统一调用 `/api/runs` | 已实现 |
| API Key | 创建、列表、删除、哈希存储 | 已实现 |
| JWT | 支持外部 JWT 鉴权 | 已实现 |
| 限流 | 按用户 QPS 限流 | 已实现 |
| 配额 | API Key 月度 token quota 拦截 | 已实现 |
| Tier 权限 | API Key `allowed_tiers` 控制 cheap/expensive | 已实现 |
| 静态规则 | 按工具、关键词、内容长度路由 | 已实现 |
| 意图分类 | 规则未命中时用分类模型判断 tier | 已实现 |
| 意图缓存 | Redis 缓存分类结果 | 已实现 |
| 负载均衡 | 同 tier 内按健康状态、连接数、权重选择 | 已实现 |
| 非流式降级 | 模型失败后尝试其他模型 | 已实现 |
| 熔断 | 连续失败后冷却摘除，冷却后恢复 | 已实现 |
| 结果缓存 | 低温度非流式请求可缓存 | 已实现 |
| 用量统计 | token、延迟、成本写入 SQLite | 已实现 |
| 配置查看 | `/api/config` 返回脱敏配置 | 已实现 |
| 管理后台 | `/admin` 内置管理页面 | 已实现 |

## 待增强

| 模块 | 需求 | 优先级 |
|------|------|--------|
| RBAC | admin/operator/viewer/app key 权限隔离 | 高 |
| 审计日志 | 记录 key 创建、删除、配置变更、管理员操作 | 高 |
| IP 白名单 | 按 key 或业务线限制来源 IP | 高 |
| 配置编辑 | 管理后台编辑模型、规则、价格配置 | 中 |
| 配置版本 | 配置修改前后 diff、回滚和审批 | 中 |
| Provider Key 轮换 | 同一 provider 多 key 轮询和失败切换 | 中 |
| 缓存策略 | 按业务线、模型、请求类型配置缓存 TTL | 中 |
| 分布式状态 | 负载、熔断、metrics 从进程内迁到 Redis/DB | 高 |
| 成本报表 | 按天、业务线、key、模型统计费用 | 高 |
| 告警 | 异常 token、成本突增、错误率升高通知 | 中 |

## 非目标

当前阶段不做：

- 模型训练和微调。
- 复杂工作流编排。
- Prompt 管理平台。
- 多租户计费系统的完整财务闭环。

这些能力可以后续作为独立子系统接入 Gateway。
